典型分析场景地图

一、测序数据预处理与比对

质控和修剪可用 @fastqc、@trimmomatic；短读长比对可用 @bwa；多序列比对可用 @mafft；HTS 文件处理可用 @samtools。

区间运算可用 @bedtools、@pybedtools、@pyranges；变异文件处理可用 @pysam、@cyvcf2、@gatk；遗传分析可用 @plink、@plink2。

三、RNA-seq 与功能富集

差异表达可用 @DESeq2 或 @pydeseq2；富集分析可用 @clusterProfiler、@gseapy；公共数据可用 @GEOparse。

四、单细胞与空间组学

单细胞主流程可用 @scanpy、@anndata；深度建模可用 @scvi-tools；RNA velocity 可用 @scvelo；空间分析可用 @squidpy、@spatialdata。

五、药物发现与化学信息学

分子处理可用 @rdkit、@openbabel；虚拟筛选可用 @pyscreener、@AutoDock Vina；DTI 和性质预测可用 @DeepPurpose。

六、机器学习与高维数据建模

传统机器学习可用 @scikit-learn；深度学习可用 @torch、@jax、@flax；降维和邻域搜索可用 @umap-learn、@faiss-cpu、@pynndescent。

七、数据格式、大数据与自动化

大型矩阵和多维数据可用 @h5py、@zarr、@xarray；并行计算可用 @dask、@joblib；API 和文件处理可用 @requests、@httpx、@pypdf。

八、可视化与报告生成

Python 绘图可用 @matplotlib、@seaborn、@altair、@bokeh；大数据渲染可用 @datashader；R 可视化可用 @ggplot2。

生物信息学任务场景地图

一、测序 reads 比对

短读长可用 @bowtie2，RNA-seq 可用 @STAR 或 @HISAT2，长读长可用 @minimap2。这一步相当于把测序碎片“归位到基因组地图”。

二、基因组与转录组组装

长读长组装可选 @Canu、@Flye、@hifiasm、@Verkko；微生物和宏基因组可选 @SPAdes、@MEGAHIT、@Unicycler；转录组从头组装可用 @Trinity。

三、基因组注释与质量控制

细菌注释可用 @Bakta、@Prokka；组装质量可用 @QUAST；MAG 质量可用 @CheckM2；多样本结果汇总可用 @MultiQC。

四、变异检测与结构变异分析

通用 VCF 处理可用 @BCFtools；长读长小变异可用 @Clair3、@Longshot、@PEPPER-DeepVariant；结构变异可用 @Sniffles。

五、RNA-seq 表达定量与融合基因

转录本定量可用 @Kallisto、@Salmon；转录本组装与定量可用 @StringTie；肿瘤融合转录本检测可用 @STAR-Fusion。

六、结构预测与蛋白设计

结构预测可用 @AlphaFold、@Boltz-2、@Chai-1；抗体/TCR 可用 @ImmuneBuilder、@RFAntibody；从头设计与序列设计可用 @RFDiffusion、@ProteinMPNN。

七、分子克隆与实验设计

引物设计可用 @design_pcr_primers_with_overhangs、@design_sanger_verification_primers；克隆可用 Gibson、Golden Gate、Gateway 相关工具；质粒信息可用 Addgene 集成工具。

八、实验 protocol 与验证分析

转化、转染、慢病毒包装、FACS、Western blot、基因编辑 amplicon PCR 等可通过对应 protocol 工具生成标准化流程，适合把湿实验步骤变成可复用的操作资产。

科研任务场景地图

一、肿瘤靶点与分子分型

优先使用 @TCGA、@cBioPortal、@DepMap。前者像“肿瘤分子档案馆”，后两者更适合做癌症基因组浏览、依赖性分析和治疗机会发现。

二、疾病机制与遗传证据

可组合 @DISGENET、@GWAS Catalog、@OpenTargets、@Monarch、@HPO。适合从疾病、表型、变异和基因证据出发，建立候选靶点证据链。

三、药物发现与老药新用

可使用 @ChEMBL、@PubChem、@Broad Repurposing Hub、@OpenFDA、@DailyMed 与 @ClinicalTrials.gov，覆盖化合物、生物活性、临床进展和安全标签。

四、基因组注释与公共测序数据

基础注释可用 @Ensembl、@NCBI Gene、@NCBI Protein；公共数据可查 @GEO、@SRA；变异解释可用 @dbSNP、@ClinVar、@gnomAD。

五、通路、互作与机制解释

可联动 @STRING、@BioGRID、@Reactome、@Gene Ontology、@MSigDB。这些数据库像“分子关系交通图”，帮助把基因列表解释成通路和网络。

六、单细胞与细胞身份注释

可使用 @CZI Cell Census、@Human Cell Atlas、@CellMarker 2.0。适合单细胞参考映射、细胞类型识别和组织细胞组成解释。

七、蛋白结构与功能域分析

结构研究可用 @PDB、@AlphaFold DB、@UniProt、@InterPro、@EMDB，覆盖实验结构、预测结构、蛋白功能和结构域信息。

八、调控元件与转录调控

可使用 @ENCODE、@ENCODE SCREEN、@JASPAR。适合增强子、启动子、转录因子 motif、染色质开放和候选调控元件分析。

场景选择地图

一、找数据与看格局

优先使用 omics-dataset-retrieval、clinicaltrials-landscape、literature-preclinical。适合项目立项、竞品调研、疾病领域扫描和证据地图构建。

二、做基础组学分析

bulk RNA-seq 可选 bulk-rnaseq-counts-to-de-deseq2 或 bulk-rnaseq-differential-expression-multi-method；单细胞可选 Scanpy、Seurat、轨迹推断、细胞通讯和空间转录组 skills。

三、解释机制与调控关系

可组合使用 functional-enrichment-from-degs、coexpression-network、grn-pyscenic、chip-atlas-target-genes 和 upstream-regulator-analysis。

四、靶点发现与验证

遗传证据可用 GWAS/TWAS、孟德尔随机化、PRS 和 Open Targets；实验验证可衔接 PCR 引物设计、sgRNA 设计、C. elegans RNAi、蛋白结构收集等 skills。

五、标志物与临床转化

可使用 lasso-biomarker-panel、survival-analysis-clinical、serum-proteomics-treatment-response，从候选分子走向风险分层、疗效预测和验证面板。
