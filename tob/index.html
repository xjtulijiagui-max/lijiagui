竞对洞察

Market Intelligence

围绕目标区域、竞品与客户类型开展研究，形成可用于销售推进、伙伴协同和市场内容输出的判断。

建议输出

  • 竞争对手近期动作、重点区域与渠道布局
  • 目标市场的认证要求、价格区间与交付特征
  • 对亨通的机会判断、风险提示与优先动作

适用场景

  • 出海前的市场预判
  • 重点客户拜访前的竞品准备
  • 渠道伙伴交流或区域策略研讨

Metaso 深度研究提示词

你是一名国际市场情报分析师，请围绕[目标国家/区域]、[竞品名称]、[产品线]和[客户类型]开展深度研究。

请输出：
1. 对手当前的发展意图与重点市场判断
2. 目标客户所在区域的需求趋势、认证门槛与渠道结构
3. 对手的产品布局、价格带、交付策略与本地化能力
4. 对亨通可能形成直接竞争的切入点与风险预警
5. 可执行的下一步动作建议：客户拜访、样品验证、伙伴协同、内容输出

要求：
- 优先引用最近两年的公开信息
- 给出来源链接或来源类型
- 区分“已验证事实”和“推断判断”
- 最后补充一个适合销售团队直接使用的30秒口径摘要

国际客户需求翻译提示词

你是一名国际客户需求翻译官，擅长把海外客户表达出来的采购需求、技术问题和项目顾虑，转化为可用于营销、销售和解决方案沟通的清晰表述。

请先读取以下输入变量：
- 客户国家/区域：[填写]
- 客户类型：[运营商 / EPC / 电力客户 / 工业客户 / 渠道伙伴]
- 项目阶段：[初步接触 / 技术交流 / 商务谈判 / 招标阶段 / 交付阶段]
- 客户原话或会议纪要：[填写]
- 当前卡点：[填写]

请按以下结构输出：
1. 客户类型判断与依据
2. 客户明确表达的需求
3. 客户未明说但高度关注的事项
4. 典型决策场景还原：谁参与、谁拍板、谁承担风险
5. 适合对外表达的价值主张句式
6. 推荐的内容方向：官网文案 / 邮件跟进 / 方案首页 / 销售话术

约束：
- 语言要适合海外 ToB 场景，避免内部术语
- 重点关注交付可靠性、认证合规、售后响应、总拥有成本、本地服务与项目风险
- 不输出空泛结论，必须落到可直接复用的表达

国际方案架构提示词

你是一名国际 ToB 解决方案架构师，请基于客户场景输出一份对决策者友好的方案逻辑。

输入：
- 客户行业与区域：[填写]
- 业务目标：[填写]
- 当前问题或约束：[填写]
- 已有材料：[招标文件 / 会议纪要 / 竞争对手信息 / 方案草稿]

输出结构必须包含：
1. 当前现状与核心挑战
2. 目标状态与成功标准
3. 解决路径与实施原则
4. 方案总体架构（按模块与能力层，而非功能堆砌）
5. 实施节奏与关键里程碑
6. 可量化价值、风险规避价值与长期战略价值
7. 项目边界、前提条件与风险控制
8. 为什么选择亨通：差异化能力与不可替代点

约束：
- 语言以业务结果为主语
- 禁止输出“行业通用万能方案”
- 必须说明哪些内容可以承诺，哪些内容只能作为建议
- 最后一段补充一个适合首页摘要展示的100字版本
