发布 HTML 到 GitHub Pages 完整流程

这份文档记录了如何将本地 HTML 文件发布到 GitHub Pages 的完整操作步骤，包括 Git 配置、文件推送和 GitHub Pages 设置。

一、初始请求

用户请求：
参考这个教程帮我把这个html文件发布为github pages C:\Users\xjtul\Downloads\ai_api_token_tools_guide.html

二、环境检查

Claude Code 操作：
检查 Git 环境和 GitHub CLI
$ git --version && gh --version git version 2.53.0.windows.2 gh: command not found

结果：有 Git 但没有 GitHub CLI，继续使用 Git 命令行方式。

三、准备本地仓库

创建项目目录并复制文件：
$ mkdir -p /tmp/ai-api-token-guide $ cp "ai_api_token_tools_guide.html" /tmp/ai-api-token-guide/index.html $ git init Initialized empty Git repository

四、Git 配置

首次提交遇到错误：
$ git commit -m "Initial commit" Error: Author identity unknown
用户提供 GitHub 地址：
https://github.com/xjtulijiagui-max/lijiagui
配置 Git 用户信息：
$ git config user.name "xjtulijiagui-max" $ git config user.email "xjtulijiagui-max@users.noreply.github.com" $ git commit -m "Initial commit" [master (root-commit) cd8c1dc] Initial commit 1 file changed, 454 insertions(+)

五、推送到 GitHub

配置远程仓库并推送：
$ git branch -M main $ git remote add origin https://github.com/xjtulijiagui-max/lijiagui.git $ git push -u origin main * [new branch] main -> main branch 'main' set up to track 'origin/main'.
✓ 文件已成功推送到 GitHub！

六、启用 GitHub Pages

配置步骤：
1. 打开仓库设置页面：https://github.com/xjtulijiagui-max/lijiagui/settings/pages 2. Source 选择：Deploy from a branch 3. Branch 选择：main 和 / (root) 4. 点击 Save 按钮 5. 等待部署完成（2-5分钟）
用户确认：
已经配置完毕

七、发布完成

🎉 发布成功！

页面地址：https://xjtulijiagui-max.github.io/lijiagui/

现在任何人都可以通过这个链接访问页面。

八、关键要点总结

Git 配置

  • 首次使用 Git 需要配置用户名和邮箱
  • 可以使用 GitHub 的 noreply 邮箱保护隐私

文件命名

  • GitHub Pages 首页必须是 index.html
  • 其他页面使用 filename.html 格式

分支管理

  • GitHub Pages 默认从 main 分支部署
  • 也可以使用 gh-pages 分支

部署时间

  • 首次部署通常需要 2-5 分钟
  • 后续更新一般在 1-2 分钟内完成
